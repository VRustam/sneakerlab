# Screenshot Capture Checklist

No screenshots are committed until they are captured from a real local, preview, or production-quality run. Do not replace these with generated images.

Use an account and generic product data that are safe to publish. Remove browser chrome, addresses, tokens, and personal information before saving captures.

- [ ] Desktop web home and catalog at a wide viewport.
- [ ] Mobile web catalog with filters and navigation open.
- [ ] Product detail with gallery, variant controls, and the Pulse Layer interactive 3D preview.
- [ ] Product detail using the image fallback (a model-less product or deliberate non-production failed model).
- [ ] Signed-in favorites, cart, demo checkout disclosure, and order history.
- [ ] Admin product editor showing image/model media controls, with no secret values visible.
- [ ] Flutter home/shop and product detail with touch 3D viewer on Android.
- [ ] Flutter product fallback on Android or iOS.
- [ ] Flutter cart, demo checkout, orders, and account screens.

Suggested filenames: `web-catalog-desktop.png`, `web-product-3d.png`, `web-admin-media.png`, `mobile-product-3d-android.png`, and `mobile-product-fallback-ios.png`.
